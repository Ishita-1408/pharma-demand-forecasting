# 💊 Pharmaceutical Demand Forecasting System



## 📌 Overview

A time-series machine learning system that forecasts monthly pharmaceutical product demand. The system supports **inventory planning**, reduces **stockouts**, and improves **production scheduling** across multiple regions.

**Key Results:**
- Forecasts demand for 8 pharmaceutical products across 5 regions (North, South, East, West, Central)
- SARIMAX achieved an average MAPE of ~7.6% across all products on a 6-month holdout, and was the best-performing model on 7 of 8 products
- Best individual result: Clonazap-0.5 at 2.28% MAPE (SARIMAX)
- Identifies seasonal demand peaks (winter antidepressant spikes, Q4 hospital restocking, October Diwali dip)
- Interactive Streamlit dashboard with downloadable forecast CSVs

## 🚀 Setup & Run (Local)

### 1. Clone the repository
```bash
git clone https://github.com/Ishita-1408/pharma-demand-forecasting.git
cd pharma-demand-forecasting
```

### 2. Create a virtual environment (recommended)
```bash
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```
If you hit build errors on Windows (common with `numpy`/`pandas` on newer Python versions), make sure `requirements.txt` has **no version pins** — pip will then fetch pre-built wheels instead of compiling from source.

### 4. Run the notebooks in order
Open the project folder in VS Code or Jupyter, then run:
```
01_EDA.ipynb → 02_Preprocessing.ipynb → 03_ARIMA_Model.ipynb → 04_Prophet_Model.ipynb → 05_Evaluation.ipynb
```
Each notebook saves outputs to `data/processed/` and `reports/figures/`, which subsequent notebooks depend on — so run them strictly in order, top to bottom.

### 5. Launch the dashboard
```bash
streamlit run dashboard/app.py
```
This opens automatically at `http://localhost:8501`.

## 📈 Model Performance

Evaluated on a 6-month holdout (Jun–Dec 2024), using MAE, RMSE, and MAPE.

| Product | SARIMAX MAE | SARIMAX RMSE | SARIMAX MAPE | Prophet MAPE | Best Model |
|---|---:|---:|---:|---:|:---:|
| Clonazap-0.5 | 23.3 | 34.2 | **2.28%** | 6.40% | SARIMAX |
| Fluox-20 | 49.0 | 58.4 | **3.16%** | 137.39% | SARIMAX |
| Psychoril-25 | 47.3 | 54.1 | **3.60%** | 28.49% | SARIMAX |
| Alprazone-0.25 | 48.7 | 56.9 | **4.06%** | 11.16% | SARIMAX |
| Escitab-10 | 92.3 | 123.9 | **4.67%** | 29.06% | SARIMAX |
| Risperidon-2 | 42.9 | 47.8 | **5.64%** | 56.99% | SARIMAX |
| Quetinil-100 | 76.7 | 83.9 | **11.09%** | 21.86% | SARIMAX |
| Lithosun-SR | 216.2 | 258.1 | 25.08% | **22.29%** | Prophet |

**Average MAPE across all products: ~7.6% (SARIMAX)**
**Model selection: SARIMAX wins on 7/8 products; Prophet wins on 1/8 (Lithosun-SR)**

### Metric Definitions
- **MAE (Mean Absolute Error)** — average absolute difference between predicted and actual units sold. Scale-dependent, expressed in units.
- **RMSE (Root Mean Squared Error)** — square root of average squared errors; penalizes large misses more heavily than MAE.
- **MAPE (Mean Absolute Percentage Error)** — average absolute error as a percentage of actual demand. Scale-independent, allowing comparison across products with different volumes. MAPE < 10% is generally considered excellent for pharmaceutical demand forecasting.

### Notes on Results
SARIMAX outperformed Prophet on 7 of 8 products in this evaluation. This is consistent with forecasting literature showing Prophet typically needs longer series (2–3+ years of monthly data) to learn seasonal patterns reliably; with only 30 training months, SARIMAX's structured autoregressive approach generalized better here. Prophet's error was especially high on lower-volume or more volatile products (e.g. Fluox-20, Risperidon-2), suggesting its flexible trend/seasonality components overfit on short series. Lithosun-SR was the one exception, where Prophet (22.29% MAPE) slightly outperformed SARIMAX (25.08% MAPE) — both results indicate this product has an irregular demand pattern that neither model captures particularly well, flagging it as a candidate for a different modelling approach (e.g., ensemble methods or a longer training window) in future iterations.

## 🔍 Key Findings

1. **Antidepressants (Escitab-10, Fluox-20)** show strong winter seasonality — demand peaks ~18% above average in Nov–Feb.
2. **October Diwali dip** affects all products (~8% drop) due to logistics slowdown.
3. **North region** accounts for ~30% of total demand — priority region for inventory.
4. **Q4 hospital restocking** creates a predictable demand spike in Nov–Dec.
5. **Quetinil-100** was the only product with recorded stockout events in the historical data — flagged for safety stock review.
6. **Lithosun-SR** is the hardest product to forecast for both models (MAPE > 22%), indicating genuinely irregular demand that may benefit from a different modelling approach (e.g. ensemble methods or longer training history) rather than relying on a single SARIMAX/Prophet choice.
